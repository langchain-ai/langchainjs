# Build Plugins

The LangChain build tool uses a set of plugins that are documented in this file.

## LangChain Secrets Plugin

A simple Rolldown plugin that scans for `lc_secrets` patterns in TypeScript files and generates corresponding TypeScript interfaces.

### Usage

```typescript
import { lcSecretsPlugin } from './lc-secrets-plugin.js'

await build({
    // ... your build config
    inputOptions: {
        plugins: [
            lcSecretsPlugin({
                enabled: true,              // Enable/disable scanning
                strict: true,               // Fail build on validation errors
                outputPath: 'load/import_type.ts', // Output file path
                excludePatterns: ['.test.ts'] // Files to exclude
            })
        ]
    }
})
```

### What it detects

The plugin looks for this pattern in your TypeScript classes:

```typescript
class MyProvider {
  get lc_secrets(): { [key: string]: string } {
    return {
      API_KEY: "OPENAI_API_KEY",        // ✅ Valid
      SECRET_TOKEN: "MY_SECRET_TOKEN",  // ✅ Valid
      "api-key": "INVALID_KEY",         // ❌ Must be uppercase
      "SPACED KEY": "ANOTHER_KEY",      // ❌ No whitespace allowed
    };
  }
}
```

### Generated output

Creates `src/load/import_type.ts`:

```typescript
// Auto-generated by lc-secrets plugin. Do not edit manually.

export interface OptionalImportMap {}

export interface SecretMap {
  OPENAI_API_KEY?: string;
  MY_SECRET_TOKEN?: string;
}
```

### Environment variables

- `SKIP_SECRET_SCANNING=true` - Disable plugin completely
- `NODE_ENV=production` - Enable strict validation

### Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `enabled` | boolean | `true` | Enable/disable secret scanning |
| `strict` | boolean | `true` | Throw errors on validation failures |
| `outputPath` | string | `"load/import_type.ts"` | Output file path relative to src/ |
| `excludePatterns` | string[] | `[".test.ts", "test.ts", ".spec.ts", "spec.ts"]` | File patterns to exclude |
