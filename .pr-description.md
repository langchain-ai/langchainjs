## Summary

Adds support for OpenAI's [Tool Search](https://platform.openai.com/docs/guides/tools-tool-search) feature to `@langchain/openai`. This is the JS port of [langchain-ai/langchain#35582](https://github.com/langchain-ai/langchain/pull/35582).

Tool Search lets models dynamically discover and load tools on-demand from a large pool via the Responses API, rather than requiring all tool definitions in every request. Tools marked with `defer_loading: true` have their schemas deferred — the model issues a `tool_search_call` to find matching tools, receives a `tool_search_output` with discovered definitions, and then makes the actual function call.

## Changes

### `@langchain/core`

**Block translator** (`messages/block_translators/openai.ts`)
- `tool_search_call` → `server_tool_call` with `name: "tool_search"`, preserving `execution`, `status`, and `call_id` in extras.
- `tool_search_output` → `server_tool_call_result` with status mapping (`completed` → `success`, `failed` → `error`), tools in output, and `name: "tool_search"` in extras.

### `@langchain/openai`

**New `tools.toolSearch()` factory** (`tools/toolSearch.ts`)
- Creates a `{ type: "tool_search" }` server tool definition, following the same pattern as `tools.webSearch()`.
- Supports `execution` (`"server"` | `"client"`), `description`, and `parameters` options for client-executed search.

**`defer_loading` propagation via extras** (`chat_models/base.ts`)
- `bindTools()` now checks for `extras: { defer_loading: true }` on LangChain tools and merges `defer_loading: true` into the converted OpenAI function tool definition. This mirrors how the Anthropic integration handles the same field via `AnthropicToolExtrasSchema`.

**Responses API tool formatting** (`chat_models/responses.ts`)
- `_reduceChatOpenAITools()` now forwards extra keys (like `defer_loading`) from completions-format tool definitions to Responses API function tools. Previously only `type`, `function.*`, and `strict` were copied.

**Response parsing** (`converters/responses.ts`)
- Streaming: `convertResponsesDeltaToChatGenerationChunk` includes `tool_search_call` and `tool_search_output` in the list of items that emit `tool_outputs` chunks.
- Round-trip: `tool_search_call` and `tool_search_output` items in `response_metadata.output` are passed back verbatim to the API on subsequent turns.

### Usage

```typescript
import { ChatOpenAI, tools } from "@langchain/openai";
import { tool } from "@langchain/core/tools";
import { z } from "zod";

const model = new ChatOpenAI({ model: "gpt-5.3" });

const getWeather = tool(
  async (input) => `Weather in ${input.location}: sunny, 72°F`,
  {
    name: "get_weather",
    description: "Get the current weather for a location",
    schema: z.object({ location: z.string() }),
    extras: { defer_loading: true },
  }
);

const response = await model.invoke("What is the weather in SF?", {
  tools: [tools.toolSearch(), getWeather],
});
```
