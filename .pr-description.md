## Summary

Adds string-model constructor overloads to all chat model provider integrations, enabling a simpler shorthand syntax:

```ts
// Before (still works)
const model = new ChatOpenAI({ model: "gpt-4o", temperature: 0.2 });

// After (new shorthand)
const model = new ChatOpenAI("gpt-4o", { temperature: 0.2 });
```

This makes the most common constructor pattern — passing a model name — more ergonomic across all 18 provider packages.

---

## Changes

### Pattern

Every chat model constructor now accepts two overload signatures:

1. `constructor(model: string, fields?: Omit<Fields, "model">)` — string shorthand
2. `constructor(fields?: Fields)` — existing object form (unchanged behavior)

The implementation resolves which form was used at runtime via `typeof modelOrFields === "string"`.

### Shared helpers

- `getChatOpenAIModelParams()` in `@langchain/openai` base — shared by OpenAI Completions, Responses, and the unified `ChatOpenAI` class
- `getAzureChatOpenAIParams()` in `@langchain/openai` Azure common — shared by all three Azure chat model variants (maps the string to `deploymentName` / `azureOpenAIApiDeploymentName` / `model`)

### Tests added

- Unit tests (`*.test.ts`) verifying the shorthand constructor for each provider
- Type tests (`*.test-d.ts`) for Cloudflare and Yandex confirming overload type safety (including `@ts-expect-error` for invalid arguments)

### No breaking changes

The existing `constructor(fields?)` signature is preserved as-is for every class. All existing call sites remain valid.
