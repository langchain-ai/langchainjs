## Summary

This PR adds support for xAI's new server-side agentic tools and deprecates the old `live_search` tool.

xAI has introduced a new **agentic tool calling API** that replaces the deprecated Live Search API. The new tools are server-side tools executed by the xAI API as part of an autonomous reasoning loop, enabling complex multi-step research and analysis.

---

## Changes

### New Tools (`libs/providers/langchain-xai/src/tools/`)

| Tool | File | Description |
|------|------|-------------|
| `xaiWebSearch()` | `web_search.ts` | Web search with `allowedDomains`, `excludedDomains`, `enableImageUnderstanding` |
| `xaiXSearch()` | `x_search.ts` | X (Twitter) search with handle filtering, date ranges, and media understanding |
| `xaiCodeExecution()` | `code_execution.ts` | Python code execution for calculations and analysis |
| `xaiCollectionsSearch()` | `collections_search.ts` | Search through uploaded knowledge bases (vector stores) |

### Deprecation

- **`xaiLiveSearch()`** is now deprecated with `@deprecated` JSDoc annotation and migration guidance pointing users to `xaiWebSearch()` and `xaiXSearch()`

### ChatXAIResponses Updates

- Added `tools` property to constructor options (`ChatXAIResponsesInput`)
- Added `tools` property to call options (`ChatXAIResponsesCallOptions`)
- Updated `invocationParams()` to pass tools to the API
- Updated `XAIResponsesWebSearchTool` type with correct filtering options (`allowed_domains`, `excluded_domains`, `enable_image_understanding`)

## Usage

```typescript
import { ChatXAIResponses, tools } from "@langchain/xai";

const llm = new ChatXAIResponses({
  model: "grok-4-1-fast",
});

// Web search
const result = await llm.invoke("What are the latest AI developments?", {
  tools: [
    tools.xaiWebSearch({ allowedDomains: ["arxiv.org"] }),
    tools.xaiXSearch({ allowedXHandles: ["elonmusk"] }),
    tools.xaiCodeExecution(),
  ],
});
```
